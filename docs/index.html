keysme-ai

BYOK (Bring Your Own Key) LLM 설정 registry + headless React 훅.
프로바이더와 프리셋을 관리하고, 암호화 토큰으로 앱 간에 키를 이동합니다. LLM 호출은 하지 않습니다.

pnpm add @keysme-ai/core @keysme-ai/react

시작하기

스토어 만들고 앱 감싸기

import { createKeysmeStore, localStorageAdapter } from '@keysme-ai/core'
import { KeysmeProvider } from '@keysme-ai/react'

const store = createKeysmeStore({ storage: localStorageAdapter() })

function App() {
  return (
    <KeysmeProvider store={store}>
      <Settings />
      <Chat />
    </KeysmeProvider>
  )
}

프로바이더와 프리셋 등록

// 프로바이더: 엔드포인트 + 키 (인프라)
await store.providers.add({
  id: crypto.randomUUID(),
  name: 'OpenAI',
  baseURL: 'https://api.openai.com/v1',
  apiKey: 'sk-...',
})

// 프리셋: 모델 + 파라미터 (재사용 설정 묶음)
await store.presets.add({
  id: 'main',
  name: '기본',
  providerId: '...',
  model: 'gpt-5',
  params: { temperature: 0.7 },
})

// 활성 프리셋 선택
await store.active.set({ presetId: 'main' })

훅으로 설정 꺼내 쓰기

import { usePresetConfig } from '@keysme-ai/react'

function Chat() {
  const { config } = usePresetConfig('main')
  // config: { baseURL, apiKey, protocol, model, params, headers }
}

핵심 개념

개념역할
ProviderLLM 엔드포인트 인스턴스. baseURL, apiKey, headers, protocol(openai | anthropic | gemini)
Preset재사용 가능한 설정 묶음. providerId + model + 생성 파라미터
ActiveSelection현재 선택된 프리셋
ResolvedConfigProvider + Preset을 합쳐 SDK에 바로 넘길 수 있게 펼친 것
model과 baseURL은 항상 필수입니다. keysme는 기본 모델이나 기본 엔드포인트를 가정하지 않습니다.
3

LLM 호출하기

keysme는 호출하지 않습니다. config.protocol이 어떤 SDK/wire format을 쓸지 알려주고, getProtocol()이 프로토콜별 잡일을 처리합니다. 

import { getProtocol } from '@keysme-ai/core'
import { toClientConfig } from '@keysme-ai/core/adapters'

// 클라이언트 설정만 추출: { baseURL, apiKey, headers }
const client = new OpenAI(toClientConfig(config))

// 파라미터를 해당 wire format 이름으로 변환
const params = getProtocol(config.protocol).toRequestParams(config.params)
await client.chat.completions.create({ model: config.model, messages, ...params })

getProtocol()이 주는 것

  • authHeaders(apiKey): Bearer / x-api-key / x-goog-api-key
  • parseModels(json): GET /models 응답에서 모델 id 추출
  • toRequestParams(params): 안정적인 이름 변환 (top_p → topP 등)

자주 바뀌는 설정은 extraBody로

reasoning, thinking처럼 프로바이더가 자주 바꾸는 옵션은 매핑하지 않습니다. params.extraBody에 넣으면 그대로 통과됩니다. 현재 권장 형태는 getProtocol JSDoc을 참고하세요.

앱 간 키 이동 (Export / Import)

keysme의 핵심 기능입니다. 유저가 정한 비밀번호로 설정 전체를 잠근 keysme1. 토큰 하나로, keysme 기반 앱 어디로든 설정을 옮깁니다. 

import { encryptExport, decryptExport, isExportToken } from '@keysme-ai/core'

// 내보내기: 유저 passphrase로 잠근 토큰 생성
const data = await store.export({ includeSecrets: true })
const token = await encryptExport(data, passphrase)
// "keysme1.eyJ2IjoxLC..." 문자열 하나

// 가져오기: 같은 passphrase로 풀기
await store.import(await decryptExport(token, passphrase), { merge: true })

전송수단은 자유

  • .keysme 파일로 저장
  • 텍스트로 복사해 붙여넣기

둘 다 같은 토큰입니다. 문자열 하나라 어떤 방식으로든 옮길 수 있습니다.

merge 옵션

  • merge: true: 기존에 합침, id 중복 시 덮어쓰기
  • merge: false(기본): 기존 삭제 후 교체

isExportToken(text)으로 암호화 토큰과 평문 JSON을 구분합니다.

React 훅

훅용도
useProviders()프로바이더 목록 + add/update/remove
usePresets()프리셋 목록 + add/update/remove
useActiveSelection()현재 선택 프리셋 조회/변경
usePresetConfig(id)특정 프리셋의 ResolvedConfig
useActiveConfig()활성 프리셋의 ResolvedConfig
useModels(providerId)프로바이더의 모델 목록 fetch/캐시
useModelCapabilities(model)모델별 지원 파라미터 조회
useProxyMode()proxy 모드 토글 (Provider.proxyBaseURL 경유)

모든 훅은 useSyncExternalStore 기반으로 필요한 slice만 구독합니다. 관련 없는 데이터가 바뀌어도 리렌더되지 않습니다.

모델 capabilities와 파라미터 검증

// 스토어 생성 시 레지스트리 주입 (선택)
const store = createKeysmeStore({
  storage: localStorageAdapter(),
  modelCapabilities: {
    'gpt-5': { excludes: ['temperature'], supports: ['reasoning_effort'] },
  },
})

// 폼에서: 지원하지 않는 파라미터는 비활성화
const caps = useModelCapabilities(model)

// 저장된 값 검증: ValidationIssue[] 반환
const issues = validateParams(config.params, caps)
레지스트리는 앱이 주입하는 데이터입니다. 새 모델이 나와도 라이브러리 업데이트를 기다릴 필요가 없습니다.
7

Storage Adapter

저장소는 get/set/remove 세 함수 규격으로 분리되어 있습니다. 기본 제공: localStorageAdapter()(웹), memoryAdapter()(테스트/SSR).

커스텀 어댑터

import type { StorageAdapter } from '@keysme-ai/core'

const myAdapter: StorageAdapter = {
  get: async (key) => { /* 저장소에서 읽어 JSON 파싱 */ },
  set: async (key, value) => { /* JSON 직렬화해 저장 */ },
  remove: async (key) => { /* 삭제 */ },
}

세 함수만 맞추면 어떤 저장소든 꽂을 수 있습니다.

에러 처리

import { KeysmeError, setKeysmeLocale } from '@keysme-ai/core'

// 앱 진입점에서 한 번. 생략하면 'en'
setKeysmeLocale('ko')

try {
  await decryptExport(token, passphrase)
} catch (err) {
  if (err instanceof KeysmeError) {
    alert(err.message)
    // '비밀번호가 틀렸거나 토큰이 손상되었습니다.'
  }
}
코드의미
MODEL_REQUIRED / BASE_URL_REQUIRED필수값 누락
DUPLICATE_ID / PROVIDER_NOT_FOUND / PRESET_NOT_FOUNDCRUD 무결성
PROVIDER_IN_USE프리셋이 참조 중인 프로바이더 삭제 시도
MODEL_FETCH_FAILED모델 목록 조회 실패 (CORS 포함)
INVALID_EXPORT_TOKEN / DECRYPT_FAILED잘못된 토큰 / 비밀번호 오류·변조

모든 코드에 en/ko 메시지가 있습니다. 에러 인스턴스 없이 코드만으로 문구가 필요하면 getErrorMessage(code, locale?)를 쓰세요.

9

보안

🔐
Export 암호화

앱 밖으로 나가는 export는 유저가 정한 passphrase에서 PBKDF2(SHA-256, 60만회)로 키를 유도해 AES-256-GCM으로 잠급니다. GCM 인증 태그가 변조와 비밀번호 오류를 DECRYPT_FAILED로 즉시 감지합니다. WebCrypto 내장 기능만 사용합니다.

💾
저장

웹에서 키는 storage adapter에 평문 저장됩니다 (BYOK 웹앱의 표준 관례). 표시할 때는 maskSecret()으로 마스킹하고, 키 없이 공유하려면 export({ includeSecrets: false })를 쓰세요.
